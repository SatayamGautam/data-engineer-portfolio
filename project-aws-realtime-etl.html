Real-Time ETL Pipeline on AWS

A streaming data pipeline that ingests events in near real-time, lands them in S3, applies transformations using AWS Glue, and loads curated data into Redshift for analytics and dashboards.

AWS S3 Lambda Glue Redshift Streaming ETL

Architecture Overview

  • Source events (e.g. app logs, transactions) pushed to an API / stream.
  • AWS Lambda validates and writes raw data to an S3 landing bucket.
  • AWS Glue jobs perform schema enforcement and transformations.
  • Curated data written to S3 processed zone in partitioned format.
  • Redshift COPY commands (or Glue ETL) load curated data into fact/dim tables.
  • Downstream BI tools connect to Redshift for near real-time dashboards.

Responsibilities

  • Designed S3 data layout (landing / raw / processed zones).
  • Implemented Glue jobs using PySpark for transformations.
  • Handled schema evolution and bad records via quarantine bucket.
  • Optimised Redshift COPY and distribution/sort keys.
  • Added CloudWatch alarms & metrics for pipeline monitoring.

Key Learnings

  • How to design near real-time ETL with decoupled components.
  • Partitioning strategy in S3 to balance query performance and small files.
  • Using Glue job bookmarks for incremental processing.

Tech Stack

  • AWS Lambda, API Gateway
  • Amazon S3
  • AWS Glue (PySpark)
  • Amazon Redshift
  • CloudWatch (logs & alarms)
